# Contributing

First of all, thank you for considering contributing to this project! 🎉

We welcome contributions of all kinds, including bug fixes, new features, documentation improvements, performance enhancements, and suggestions. Every contribution, no matter how small, helps improve the project for everyone.

Please take a few minutes to read these guidelines before getting started.

---

# Code of Conduct

By participating in this project, you agree to abide by our **Code of Conduct**. Please read the [Code of Conduct](CODE_OF_CONDUCT.md) file before contributing.

We expect all contributors to interact respectfully and help maintain a welcoming and inclusive community.

---

# Before You Start

Before opening a Pull Request, please ensure that you:

* Search existing issues to avoid creating duplicates.
* Discuss major features or architectural changes by opening an issue first.
* Read the project documentation thoroughly.
* Keep your changes focused on a single issue or feature.
* Follow the project's coding style and conventions.

---

# Getting Started

1. Fork the repository.
2. Clone your fork to your local machine.

```bash
git clone https://github.com/<your-username>/<repository>.git
```

3. Navigate to the project directory.

```bash
cd <repository>
```

4. Install the required dependencies following the project's setup instructions.

5. Verify that the project builds and runs successfully before making any changes.

---

# Creating a Branch

Always create a new branch instead of working directly on the main branch.

Example:

```bash
git checkout -b feature/add-new-component
```

or

```bash
git checkout -b fix/login-validation
```

Use descriptive branch names that clearly indicate the purpose of your changes.

---

# Making Changes

While contributing, please:

* Write clean and readable code.
* Follow existing project structure and naming conventions.
* Keep commits focused and meaningful.
* Avoid unrelated changes in the same Pull Request.
* Update documentation whenever necessary.

If you modify existing functionality, ensure your changes remain backward compatible whenever possible.

---

# Commit Message Guidelines

Write meaningful commit messages.

Examples:

```text
feat: add user profile page
fix: resolve authentication bug
docs: update installation guide
refactor: simplify API service
style: improve button spacing
```

Avoid vague commit messages such as:

```text
update
changes
fixed stuff
```

---

# Pull Request Process

Before submitting your Pull Request:

* Ensure your branch is up to date.
* Verify that the project builds successfully.
* Run all available tests.
* Update documentation if your changes require it.
* Keep the Pull Request focused on a single feature or fix.

In your Pull Request description:

* Clearly explain what was changed.
* Reference the related issue using `Fixes #<issue_number>` when applicable.
* Include screenshots for UI changes, if relevant.

---

# Reporting Bugs

When reporting a bug, please include:

* A clear description of the problem.
* Steps to reproduce the issue.
* Expected behavior.
* Actual behavior.
* Screenshots or logs (if applicable).
* Your operating system and environment details.

Providing complete information helps maintainers resolve issues more efficiently.

---

# Suggesting Features

Feature requests are always welcome.

When suggesting a new feature, please explain:

* The problem you are trying to solve.
* Your proposed solution.
* Possible alternatives you considered.
* Any additional context or examples.

This helps maintainers better evaluate the proposal.

---

# Documentation Contributions

Documentation improvements are valuable contributions.

You can help by:

* Fixing grammatical errors.
* Improving explanations.
* Adding missing examples.
* Updating outdated information.
* Enhancing setup or installation instructions.

Clear documentation benefits every contributor.

---

# Need Help?

If you have questions, feel free to open an issue or start a discussion before beginning your work.

We appreciate your time and effort in helping improve this project.

Happy Contributing! 🚀
